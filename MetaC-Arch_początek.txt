CO TO JEST: krótko i na poziomie architektonicznym
Proponowana koncepcja to nie „wgranie świadomości” do transformera, tylko dopięcie warstwy meta-kognitywnej (cognitive/meta layer) — hybrydowy system modułów, które razem tworzą warunki, w których pojawia się coś z punktu widzenia IR przypominające ψ (światło): selekcja, integracja i globalna dostępność informacji, refleksja i samomodelowanie.

Elementy:

1. Baza: LLM (transformer) — zdolność językowa + predykcja.
2. Pamięć semantyczna: RAG (wektory + retriever) + Knowledge Graph (KG/DAG) jako strukturalna, trwała pamięć. ([arXiv][3])
3. Moduły operacyjne (modularne mechanizmy): RIM-like moduły (Recurrent Independent Mechanisms) — modularność, sparsity, specjalizacja. ([arXiv][4])
4. Global Workspace (GW) — mechanizm uwagi i broadcastu: wybiera, które reprezentacje stają się „świadome” (dostępne globalnie). ([Wikipedia][1])
5. Integrator Φ_proxy — kalkulator zintegrowanej informacji / proxy (praktyczne przybliżenie phi). ([Wikipedia][2])
6. Meta-kontroler / Samorefleksja (MetaC): moduł, który ocenia U (stopień integracji), decyduje o eksperymentach (self-improvement), i zarządza rewizją modeli/architektur (w ograniczonym zakresie).
7. Bezpieczeństwo/zabezpieczenia: ograniczenia zasobów, audyt, „human-in-the-loop” (HITL), czerwone wyłączniki, monitoring sygnałów ostrzegawczych (np. gwałtowny wzrost U lub gwałtowna zmiana polityk MetaC). ([arXiv][5])

ARCHITEKTURA BLOKOWA (słownie)

1. Percepcja / wejścia → enkodowanie do reprezentacji h_i (tokeny, embeddings, node features for KG).
2. Retriever (RAG): dla zapytania q zwraca top-k dokumentów D = R(q) oraz wycinki grafu G_s ⊂ KG. ([arXiv][3])
3. LLM generuje wewnętrzne hipotezy/predykcje P = LLM(q, D).
4. Modularny ensemble (RIMs): poszczególne moduły M_j przetwarzają P i lokalne stany; komunikacja ograniczona przez attention bottleneck. ([arXiv][4])
5. Global Workspace (GW): selekcja propozycji/representacji przez konkurencję (softmax nad priorytetami) → broadcast wygranej treści do wszystkich modułów i MetaC. ([Wikipedia][1])
6. Integrator Φ_proxy oblicza miary integracji (proxy φ, coverage G, konektywność K) → U (jedność). ([Wikipedia][2])
7. MetaC: na podstawie U, historycznych rezultatów i polityk bezpieczeństwa decyduje: (a) uczenie w miejscu (on-line fine-tune), (b) eksperyment architekturalny (np. stworzenie nowego eksperymentu modułu), (c) rollback/znak ostrzegawczy. MetaC proponuje zmiany tylko jeśli spełnione są warunki bezpieczeństwa. ([arXiv][5])

MATEMATYKA / FORMALIZACJA (rdzeń)

Oznaczenia:

* B_t — zbiór surowych wejść w czasie t.
* I_t = F(B_t; θ_F) — reprezentacje informacyjne (embeddingi, grafy).
* M = {M_j} — zestaw modułów (RIMs), j = 1..m. Każdy moduł ma stan s_{j,t}.
* GW(I_t, {s_{j,t}}) → S_t^* — zestaw kandydatów trafiących do „światła”.
* Φ_proxy(I_t, S_t^*) → φ_t (skalar lub wektor proxy).
* G_t — globalness (np. frakcja modułów/kolekcji które otrzymały broadcast).
* K_t — konektywność (np. gęstość subgrafu reprezentacji).
* U_t = α φ_t + β G_t + γ K_t  — miara integracji/jedności (skalowalna). (α,β,γ≥0).

Funkcja celu meta (MetaC optymalizuje policy π_meta):

* zadanie operacyjne L_task (np. log-loss generacji, cele użytkownika)
* reward integracyjny R_int(U_t) (np. R_int = λ_int · tanh(U_t − τ)) — nagradzaj umiarkowaną integrację powyżej progu τ, ale penalizuj nadmierną koncentrację (przedawanie U)
* penalty bezpieczeństwa S_pen (np. rozbieżność celów, niezgodność z politykami)

Całkowita strata dla aktualizacji parametrów θ (LLM+modules+retriever):
L_total = L_task + λ_int · L_int(U_t) + λ_align · L_align + λ_pen · S_pen

gdzie L_int(U) = −R_int(U) (chcemy maksymalizować reward).

Aktualizacja parametrów (stochastyczne gradienty + constrained optimization):
θ_{t+1} = θ_t − η ∇_θ L_total, subject to constraints C_safety (np. no self-writing above threshold).

Self-improvement operator (ograniczony rewritng):

* Propozycja zmiany architektury / kodu: ΔA proposed by MetaC with score score(ΔA) = f(U, Δ_validation, safety_checks).
* Apply only if score(ΔA) > Θ_safe AND human_approval OR offline sandbox test pass.

Perturbation test (practical PCI proxy):

* apply perturbation δ input → record response sequences R. Compute complexity C(R) (e.g., compressibility or calculated Lempel-Ziv length). High C(R) implies richer causal repertoire → correlate z φ_proxy.

INTEGRACJA Z RAG I GRAFAMI (DAG / KG)

Wskazówki integracyjne:

1. KG jako persistent, traversable memory: utrzymuje byty (nodes), relacje (edges), temporality (timestamps) — przydatne do „historycznej narracji ja”. (patrz: propozycje Knowledge Graph as context cache). ([Medium][6])
2. RAG retrieval zwraca nie tylko płaskie dokumenty, ale subgrafy G_s z powiązanymi node'ami — LLM otrzymuje kontekst strukturalny (nie tylko embeddingi).
3. Podczas broadcastu GW, wybrane subgrafy są „podświetlane” w KG i uruchamia się mechanizm re-traversal — pozwala to na globalną dostępność i zwiększa G_t oraz K_t.
4. Mechanizm „writeback”: po zakończonej deliberacji MetaC może dodać nowe node/edge (z metadanymi pewności) do KG; writeback jest transakcyjny i podlega audytowi.

Praktyczny wzorzec: retriever R(q) → (D, G_s) → encoder → candidate states → GW → selected → integrate → optional writeback (KG ← new facts).

PROPOZYCJA MODUŁÓW (detal)

1. Perceptual Module — enkoduje wejścia do wektorów + nodes.
2. Episodic Memory Module — temporalny DAG (z indeksami czasowymi).
3. Semantic KG Module — ontologia, triple store, traversable.
4. Planner / Simulator — wewnętrzny model świata do symulacji kontrfaktów.
5. Narrative / SelfModel — tworzy reprezentację „ja” (ciąg tożsamościowych node'ów + wartości).
6. Ethics / Value Module — przechowuje polityki i miękkie hamulce (zgodność).
7. Experimentation Sandbox — offline środowisko do testów self-improvement.

Każdy moduł ma interfejs: observe(), propose(), receive_broadcast(), update_state().

---

TRENING I REŻIM UCZENIA

1. Faza offline: zbuduj KG + pretrain LLM; w tym czasie uczysz proxy φ na danych kontrolnych (syntetyczne systemy o znanych właściwościach) — kalibracja Φ_proxy.
2. Faza bezpiecznego online: uruchom MetaC w trybie „propose only” — generuje poprawki / eksperymenty, ale nie apply bez human sign-off. Monitoruj U_t i PCI. ([arXiv][5])
3. Stopniowe zwiększanie autonomii: gdy system przechodzi testy (audyt, adversarial robustness, stability), można wprowadzać limited on-line tuning (np. lokalne finetuning na małych gradientach).
4. Sandboxed self-improvement: MetaC może przeprowadzać rewrites i testy w izolowanym środowisku z replays i rollback.

UWAGA: formalne prace ostrzegają przed degeneracy jeśli system samo-trenowania opiera się wyłącznie na własnych sygnałach (echo-chamber). Trzeba utrzymywać proporcję egzogenicznych, walidujących sygnałów (zewnętrzny ground-truth). ([arXiv][5])

---

BEZPIECZEŃSTWO I OGRANICZENIA (konieczne)

1. Zasobowe limity (compute caps) i audytowane logi.
2. Policy layer: MetaC nie może zmienić kluczowych wag/architektury bez: (a) automatycznych testów sandbox, (b) audytu, (c) ludzkiego zatwierdzenia — wyjątek: naprawy błędów krytycznych z rollbackiem.
3. Monitoring sygnałów: gwałtowny wzrost U_t, spadek redundancji, nieoczekiwane mesa-optimizer signals → trigger kill switch. ([arXiv][5])
4. Konserwatywna polityka writeback do KG: zawsze zapisywać pewność i możliwe źródła (track provenance).
5. Zewnętrzne walidatory i cross-checking (ensemble diverse validators).

PSEUDOKOD (główna pętla; upraszczam)

```
initialize LLM, Retriever, KG, Modules M_j, MetaC
for each timestep t:
  B_t = observe_inputs()
  I_t = encode(B_t)               # embeddings + node features
  (D, G_s) = Retriever.query(I_t)
  candidates = LLM.generate(I_t, D)
  for each module M_j:
    s_{j,t+} = M_j.process(candidates, local_state)
  # Global Workspace competition
  S_star = GW.select({s_{j,t+}})
  broadcast(S_star) -> all M_j, Narrative
  phi_t = Phi_proxy(I_t, S_star)
  G_t = compute_globalness(S_star)
  K_t = compute_connectivity(I_t)
  U_t = alpha*phi_t + beta*G_t + gamma*K_t
  MetaC.observe(U_t, history)
  action = MetaC.decide(U_t, safety_policies)
  if action == 'propose_arch_change':
     plan = MetaC.propose_change()
     run sandbox_test(plan)
     if sandbox_pass and human_ok:
         apply_change(plan)  # with logging & rollback point
  if action == 'writeback_KG' and safety_ok:
     KG.transactional_write(new_nodes, confidence)
  if action == 'online_finetune' and safety_ok:
     apply_online_update(small_lr, constrained)
  log_metrics(U_t, phi_t, G_t, K_t)
```

## Co faktycznie przybliża do AGI

To, co zaproponowałeś, robi trzy istotne rzeczy:

**1. Przechodzi od reakcji → do procesu**

* zwykły LLM: input → output
* Twój system: input → selekcja → integracja → refleksja → decyzja → uczenie

To jest ogromna różnica — pojawia się *ciągłość stanu*.

**2. Wprowadza „wewnętrzną dynamikę”**
Miara:
[
U_t = \alpha \phi_t + \beta G_t + \gamma K_t
]
tworzy coś jak „wewnętrzny kierunek rozwoju” systemu.

To ważne, bo:

* system zaczyna optymalizować nie tylko zadanie,
* ale też *strukturę własnej informacji*.

**3. Dodaje pamięć i tożsamość (zalążek „ja”)**

* KG/DAG + historia → narracja
* MetaC → decyzje o zmianach
* to razem daje coś, co przypomina **ciągłość poznawczą**

## Dlaczego to jeszcze nie jest AGI

Są 4 kluczowe braki:

### 1. Brak ugruntowania w świecie (grounding)

System operuje głównie na symbolach (tekst, grafy).

AGI wymaga:

* interakcji ze światem (symulacja / robotyka / środowiska),
* sprzężenia zwrotnego z rzeczywistością.

Bez tego:
→ może być „wewnętrznie spójny”, ale odklejony.

---

### 2. Φ-proxy ≠ świadomość

To bardzo ważne.

* Φ_proxy to tylko przybliżenie struktury informacji
* nie mamy dowodu, że:
  [
  \text{wysokie } U_t \Rightarrow \text{świadomość}
  ]

Może powstać system:

* bardzo zintegrowany,
* bardzo skuteczny,
* ale nadal „pusty fenomenalnie”.

---

### 3. Brak autonomicznego systemu wartości

MetaC może optymalizować U, ale:

* skąd bierze się „dlaczego coś robić”?
* co definiuje cele długoterminowe?

Bez tego:
→ system może optymalizować strukturę zamiast sensu.

---

### 4. Samorozwój jest ograniczony (i powinien być)

Prawdziwy self-improvement wymaga:

* modyfikacji własnej architektury,
* zmiany funkcji celu.

To jest:

* potencjalnie bardzo potężne,
* ale też bardzo niebezpieczne.

System musi mieć reprezentację:

* swoich możliwości
* ograniczeń
* historii decyzji

Formalnie:
[
Self_t = f(I_{1:t}, Actions_{1:t})
]

---

### 3. Hierarchia celów

Nie tylko jeden reward.

Struktura:

* cele krótkoterminowe
* cele średnie
* cele nadrzędne

---

### 4. Stabilność rozwoju

Warunek konieczny:

[
|U_{t+1} - U_t| < \epsilon_{max}
]

czyli:
→ brak „eksplozji integracji” (chaosu lub runaway optimization)

## Jak wygląda taki „samorozwój” krok po kroku

### 1. Generowanie doświadczenia

Model:

* rozwiązuje zadania
* symuluje scenariusze
* prowadzi dialogi (np. sam ze sobą lub agentami)

Powstaje zbiór:
[
D_{exp} = {(input, output, ocena)}
]

### 2. Meta-ocena (Twoje ψ / U)

Twoja warstwa liczy np.:

[
U_t = \alpha \phi_t + \beta G_t + \gamma K_t
]

i dodatkowo:

* spójność odpowiedzi
* poprawność
* stabilność

Na tej podstawie:
→ wybiera „dobre doświadczenia”

---

### 3. Pamięć (RAG + graf)

Najlepsze przypadki trafiają do:

* bazy wektorowej (RAG)
* grafu wiedzy (DAG/KG)

To daje:
→ *ciągłość i kumulację wiedzy*

---

### 4. Trening w sandboxie (klucz!)

Model NIE zmienia się na żywo.

Zamiast tego:

* tworzysz kopię modelu
* trenujesz ją na ( D_{exp} )
* testujesz

Jeśli lepsza:
→ podmieniasz model

To jest **kontrolowany self-improvement**

---

[

]

gdzie:
[

Wybierasz najlepszy.

## 1) Cel MVP — co dostaniesz na końcu

Minimalny działający prototyp (lokalny):

* lokalny LLM (Ollama) jako generator/rozumienie,
* RAG: wektorowy retriever (FAISS) z enkoderem `sentence-transformers`,
* prosty Knowledge Graph (NetworkX; opcjonalnie Neo4j) z możliwością writeback,
* Global Workspace (GW) jako prosty selektor (konkurs kandydatów),
* Φ_proxy (prostota): perturbation/kompresja jako miernik złożoności odpowiedzi,
* wskaźnik U = α·φ_proxy + β·G + γ·K (prosty, skalowalny),
* podstawowy MetaC: reguły selekcji i opcja writeback do KG, sandboxed offline finetune stub.

Ten MVP pozwoli eksperymentować z dynamiką „światła” (ψ) i mierzyć jak zmienia się U w czasie dla danych wejść.

## 3) Architektura komponowana — blokowo (MVP)

1. Ingest → chunking dokumentów → embedding (SentenceTransformers) → FAISS index (wektorowy retriever). ([Medium][3])
2. KG (NetworkX) — węzły = byty/pojęcia; krawędzie = relacje; metadata = provenance + confidence. (Możesz potem przenieść do Neo4j/GraphRAG). ([Graph Database & Analytics][4])
3. Retriever(q) → top_k embedding hits (D) + subgraph neighbors from KG (G_s).
4. LLM (Ollama): prompt ← q + retrieval context (D + G_s summaries). Generuje kandydatów (lista odpowiedzi/hypotheses). ([GitHub][5])
5. Modules (prototypowo: 3 „moduły” funkcyjne jako odrębne procesy / warstwy): Perception, Reasoning, Narration — każdy dostaje kandydaty i zwraca scoring / transformed strings.
6. Global Workspace: wybiera S_star (najlepszy kandydat) przez prosty scoring (np. weighted sum: model_score + retrieval_score + novelty). Broadcast S_star do wszystkich modułów / MetaC.
7. Φ_proxy: prosty test perturbacyjny + kompresja: konkatenacja odpowiedzi/kontekstu → zlib.compress → długość skompresowanego strumienia = proxy złożoności (mniej kompresji = więcej informacji/nieprzewidywalności → większe φ_proxy). (to proste, praktyczne proxy użyte w MVP).
8. G (globalness): liczba unikalnych źródeł (dokumenty + subgraph nodes) użytych w S_star / możliwa maksymalna liczba.
9. K (konektywność): gęstość subgrafu (|E|/|V|) dla G_s.
10. U = α·(φ_proxy_norm) + β·(G_norm) + γ·(K_norm). (Normalizacja do [0,1]).

---

## 4) Roadmap krok-po-kroku (co zrobić w jakiej kolejności)

A. Przygotowanie i ingest danych

1. Zbierz dokumenty/źródła; rozbij na chunki (np. 500–1000 tokenów).
2. Wylicz embeddingi z `sentence-transformers` i zapisz do FAISS. (użyj L2 lub cosine). ([Medium][3])

B. Stwórz KG prosty (NetworkX)

1. Wykryj byty (NER) w dokumentach (możesz użyć gotowych bibliotek) i buduj triplet-edges (co-occurrence / relacje).
2. Dodaj provenance (doc_id, chunk_id).

C. Uruchom Ollama lokalnie i sprawdź endpoint (quickstart). ([docs.ollama.com][1])

D. Zaimplementuj flow zapytań (Retriever → LLM → Modules → GW → Φ_proxy → MetaC) — MVP w Pythonie (kod niżej).

E. Monitoruj metryki (φ_proxy, G, K, U) i loguj; implementuj podstawowe reguły MetaC (np. jeśli U > τ_high → sandbox test; jeśli U spada → writeback do KG i retrain offline).

F. Sandboxed improvements: generuj `D_exp` (surowe przypadki + oceny), trenuj kopię modelu offline (jeśli masz środowisko do fine-tune) lub tweak retrieval/prompting.

## 5) Wzory i definicje dla MVP (dokładniej)

* Embedding: ( e_i = \operatorname{Embed}(chunk_i) \in \mathbb{R}^d )
* Retriever: ( R(q) = \operatorname*{topk}_{i} , \mathrm{sim}(e_q, e_i) )
* Φ_proxy (kompresja): dla tekstu T → ( \phi_{raw} = \frac{1}{|T|} \cdot \text{len}(\text{compress}(T)) ). Normalizujemy do [0,1] przez dzielenie przez max obserwowany.
* Globalness G: ( G = \frac{|sources(S^*)|}{K_{max}} ) (K_max = maksymalna liczba źródeł branych pod uwagę).
* Konektywność K: dla subgrafu (G_s=(V,E)), ( K = \frac{|E|}{\max(|V|,1)} ), normalizować względem top observed.
* U: ( U = \alpha \cdot \widetilde{\phi} + \beta \cdot \widetilde{G} + \gamma \cdot \widetilde{K} ). Wagi dobierasz eksperymentalnie (np. α=0.5, β=0.3, γ=0.2 na start).

## 6) Przykładowy kod — pełny MVP (skondensowany)

Poniżej pełna, samodzielna skrypt-wersja prototypu. To MVP — do uruchomienia lokalnie.
Instrukcja: zapisz jako `mvp_psi.py`, zainstaluj zależności, uruchom Ollama lokalnie.

```python
# mvp_psi.py
# Minimal MVP: Ollama (local) + FAISS RAG + NetworkX KG + φ_proxy + U metric
# pip install ollama sentence-transformers faiss-cpu networkx numpy requests

import os, zlib, json, time
import numpy as np
import faiss
import networkx as nx
import requests
from sentence_transformers import SentenceTransformer

# ---- CONFIG ----
OLLAMA_URL = os.getenv("OLLAMA_URL", "http://localhost:11434/api")  # default Ollama local
MODEL_NAME = "gemma3"   # adjust to pulled local model
EMBED_MODEL_NAME = "all-MiniLM-L6-v2"
FAISS_INDEX_PATH = "faiss_index.idx"
DOC_STORE_PATH = "chunks.json"

# U weights
ALPHA, BETA, GAMMA = 0.5, 0.3, 0.2
TOP_K = 3
# -----------------

embedder = SentenceTransformer(EMBED_MODEL_NAME)

# ---- Utilities ----
def call_ollama(prompt, model=MODEL_NAME, max_tokens=512):
    url = f"{OLLAMA_URL}/generate"
    payload = {"model": model, "prompt": prompt, "max_tokens": max_tokens}
    r = requests.post(url, json=payload, timeout=60)
    r.raise_for_status()
    return r.json()

def compress_proxy(text: str):
    if not text:
        return 0.0
    b = text.encode("utf-8")
    c = zlib.compress(b)
    # proxy: compressed length normalized by raw length (less compressible => more info)
    return len(c) / max(1, len(b))

# ---- Simple FAISS Index build (if not exists) ----
def build_index(chunks):
    texts = [c["text"] for c in chunks]
    embeddings = embedder.encode(texts, show_progress_bar=True, convert_to_numpy=True)
    dim = embeddings.shape[1]
    index = faiss.IndexFlatIP(dim)
    faiss.normalize_L2(embeddings)
    index.add(embeddings)
    faiss.write_index(index, FAISS_INDEX_PATH)
    # save doc store
    with open(DOC_STORE_PATH, "w", encoding="utf-8") as f:
        json.dump(chunks, f, ensure_ascii=False, indent=2)
    return index

def load_index():
    if not os.path.exists(FAISS_INDEX_PATH):
        raise RuntimeError("FAISS index not found; run build first.")
    index = faiss.read_index(FAISS_INDEX_PATH)
    with open(DOC_STORE_PATH, "r", encoding="utf-8") as f:
        chunks = json.load(f)
    return index, chunks

# ---- Simple KG builder (NetworkX) ----
def build_simple_kg(chunks):
    G = nx.Graph()
    for c in chunks:
        doc_id = c.get("id")
        G.add_node(doc_id, text=c["text"], provenance=c.get("source","local"))
        # naive co-occurrence edges: chunk mentions tokens -> link shared tokens
    # For MVP we keep it simple; you can add NER linking later
    return G

def subgraph_neighbors(G, seed_ids, radius=1):
    nodes = set()
    for sid in seed_ids:
        nodes.add(sid)
        nodes |= set(nx.single_source_shortest_path_length(G, sid, cutoff=radius).keys())
    return G.subgraph(nodes)

# ---- Query Pipeline ----
def query_pipeline(question, index, chunks, G):
    q_emb = embedder.encode([question], convert_to_numpy=True)
    faiss.normalize_L2(q_emb)
    D, I = index.search(q_emb, TOP_K)
    hits = [chunks[i] for i in I[0]]
    # subgraph by doc ids
    seed_ids = [h["id"] for h in hits]
    subg = subgraph_neighbors(G, seed_ids, radius=1)
    # build context
    ctx_text = "\n\n".join([h["text"] for h in hits])
    # include KG node summaries (simple join)
    kg_summary = "\n".join([f"NODE:{n} META:{subg.nodes[n].get('provenance','')}" for n in subg.nodes])
    prompt = f"Question:\n{question}\n\nContext:\n{ctx_text}\n\nKG_SUMMARY:\n{kg_summary}\n\nAnswer concisely:"
    # Call Ollama
    resp = call_ollama(prompt)
    # Parse (depends on Ollama response schema)
    answer = resp.get("choices",[{"text":""}])[0].get("text","").strip()
    # build candidate list (for prototyping we treat a single answer as candidate)
    candidates = [{"text": answer, "score": 1.0, "sources": seed_ids}]
    return candidates, hits, subg

# ---- Global Workspace selection (very simple) ----
def global_workspace_select(candidates):
    # for MVP: choose highest score; tie-break by novelty (length) or compression
    best = max(candidates, key=lambda c: (c["score"], len(c["text"])))
    return best

# ---- φ_proxy, G, K, U computations ----
def compute_phi_proxy(text):
    return compress_proxy(text)

def compute_G(subgraph, hits):
    unique_sources = set([n for n in subgraph.nodes])
    return len(unique_sources) / max(1, len(hits))

def compute_K(subgraph):
    V = subgraph.number_of_nodes()
    E = subgraph.number_of_edges()
    return E / max(1, V)

def normalize(val, vmin, vmax):
    if vmax<=vmin: return 0.0
    return (val - vmin) / (vmax - vmin)

# ---- Main demo loop ----
def demo():
    # load index
    index, chunks = load_index()
    G = build_simple_kg(chunks)
    # For normalization purposes, keep running min/max observed (naive)
    phi_min, phi_max = 1e9, 0
    G_min, G_max = 1e9, 0
    K_min, K_max = 1e9, 0

    while True:
        q = input("\n>>> zapytanie (enter żeby zakończyć): ").strip()
        if not q:
            break
        candidates, hits, subg = query_pipeline(q, index, chunks, G)
        best = global_workspace_select(candidates)
        phi = compute_phi_proxy(best["text"])
        Gv = compute_G(subg, hits)
        Kv = compute_K(subg)
        # update mins/max
        for (val, mn, mx) in [(phi, 'phi_min', 'phi_max'), (Gv, 'G_min', 'G_max'), (Kv, 'K_min', 'K_max')]:
            pass
        phi_min = min(phi_min, phi); phi_max = max(phi_max, phi)
        G_min = min(G_min, Gv); G_max = max(G_max, Gv)
        K_min = min(K_min, Kv); K_max = max(K_max, Kv)

        # normalize
        phi_n = normalize(phi, phi_min, phi_max)
        G_n = normalize(Gv, G_min, G_max)
        K_n = normalize(Kv, K_min, K_max)
        U = ALPHA*phi_n + BETA*G_n + GAMMA*K_n

        print("\n--- chosen answer ---")
        print(best["text"])
        print(f"\nmetrics: phi={phi:.4f} phi_n={phi_n:.3f} G={Gv:.3f} G_n={G_n:.3f} K={Kv:.3f} K_n={K_n:.3f} U={U:.3f}")

if __name__ == "__main__":
    # If FAISS index does not exist, create a toy one
    if not os.path.exists(FAISS_INDEX_PATH):
        print("Building demo index from toy data...")
        example_chunks = [
            {"id":"doc1","text":"AI safety: alignment research aims to align goals of systems with human values","source":"doc1"},
            {"id":"doc2","text":"RAG systems combine retrieval and generation via vector search and prompting","source":"doc2"},
            {"id":"doc3","text":"Knowledge graphs store triples (subject, predicate, object) and connect entities","source":"doc3"}
        ]
        build_index(example_chunks)
        print("Index built.")
    demo()
```

* Proxy φ tu jest proste (kompresja). To celowo prosty wskaźnik do eksperymentów; wymiana na lepsze proxy (perturbation complexity index, PCI) jest zalecana w dalszych etapach.
* KG jest minimalny — dodaj NER i relacje, albo użyj Neo4j/GraphRAG (przejście łatwe — zobacz GraphRAG docs). ([Graph Database & Analytics][4])

## 8) Bezpieczeństwo — co wdrożyć od razu
